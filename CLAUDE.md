## Architecture

This is a **React Native TurboModule** library (New Architecture) for thermal receipt printing over Bluetooth, scaffolded with `create-react-native-library`.

### Structure

- `src/` — TypeScript source; `index.tsx` exports the public API, `NativeReceiptPrinter.ts` defines the `Spec` interface and retrieves the native module via `TurboModuleRegistry.getEnforcing`
- `android/src/main/java/com/receiptprinter/` — Kotlin TurboModule implementation (`ReceiptPrinterModule.kt`) and package registration (`ReceiptPrinterPackage.kt`)
- `ios/` — Objective-C++ implementation (`ReceiptPrinter.mm` / `ReceiptPrinter.h`) with JSI bridge
- `example/` — Standalone demo app (Yarn workspace) for testing against real devices/simulators
- `lib/` — Compiled output (generated by `yarn prepare`; do not edit manually)

### TurboModule pattern

The JS–native bridge works as follows:

1. `src/NativeReceiptPrinter.ts` defines the `Spec` interface (method signatures)
2. `codegenConfig` in `package.json` drives codegen to produce native specs
3. Native classes (`ReceiptPrinterModule` on Android, `ReceiptPrinter` on iOS) implement the generated spec
4. `src/index.tsx` wraps the native module and exposes the public API

### Monorepo / tooling

- **Yarn v4 workspaces** — root + `example/` app
- **Turbo** (`turbo.json`) — `build:android` and `build:ios` tasks with caching
- **Lefthook** — pre-commit runs ESLint + `tsc --noEmit` on staged files; commit-msg enforces Conventional Commits (Angular preset: `fix`, `feat`, `refactor`, `docs`, `test`, `chore`)
- **react-native-builder-bob** — compiles `src/` to `lib/` (CommonJS, ESM, TypeScript declarations)

### TypeScript

`tsconfig.json` uses strict mode with `bundler` module resolution. The path alias `react-native-receipt-printer` → `./src/index` lets the example app import the library source directly during development.
